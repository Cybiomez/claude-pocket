# Claude Pocket

Android-клиент для собственного **Claude Code** на своём сервере. Не веб-версия Claude —
телефон подключается по SSH к твоему серверу и работает с теми же сессиями, что CLI и VSCode.

```
[Android-чат] ←SSH-туннель→ [claude-pocketd на localhost] → [Claude Agent SDK]
                                                                  ↓
                                              ~/.claude/projects/<cwd>/*.jsonl
                                              (общее хранилище с CLI и VSCode)
```

## Возможности

- Список сессий сервера, продолжение любой из них (общие с CLI/VSCode, тот же session id)
- Чат со стримом ответа, вкладки для параллельных сессий
- Показ действий агента свёрнутыми строками («Читаю main.py», «Запускаю npm test»), разворот по тапу
- Кнопка прерывания текущей работы
- Слеш-команды (включая пользовательские скиллы), выбор effort и режима прав
- Usage: лимиты 5 часов / неделя, прогресс заполнения контекста
- Ответ живёт на сервере: закрыл приложение, потерял сеть — вернулся и забрал готовое

## Установка

### Сервер (любой Linux с установленным Claude Code)

```bash
curl -fsSL https://raw.githubusercontent.com/Cybiomez/claude-pocket/main/install.sh | bash
```

Скрипт ставит демон `claude-pocketd` как systemd-сервис. Демон слушает **только 127.0.0.1** —
наружу не открывается ничего нового, телефон дотягивается через SSH-туннель.

### Телефон

Установить `claude-pocket.apk` (см. релизы), ввести адрес сервера, SSH-порт, логин
и пароль или приватный ключ. Всё — токен демона приложение заберёт само.

## Устройство репозитория

- `daemon/` — демон на Node.js + `@anthropic-ai/claude-agent-sdk`: HTTP API на localhost,
  очередь сообщений в SQLite, SSE-стрим событий
- `android/` — приложение: Kotlin + Jetpack Compose, SSH-клиент внутри (JSch)
- `install.sh` — установка демона в одну команду
- `docs/decisions.md` — журнал принятых решений

## Безопасность (текущая модель)

- Транспорт — только SSH; демон снаружи невидим
- API демона закрыт Bearer-токеном (`~/.claude-pocket/token`, права 600) — от других
  локальных пользователей машины
- Приложение хранит доступы в приватном хранилище приложения
- Известные упрощения (места помечены TODO, меняются точечно):
  `StrictHostKeyChecking=no` в SSH-клиенте; режим по умолчанию `bypassPermissions`
  (агент не спрашивает разрешений — как `--dangerously-skip-permissions`)
