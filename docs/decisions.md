# Журнал решений — Claude Pocket

Сессия 17-07-2026, автономная реализация пайплайна №4.

## Разведка (задача 2 пайплайна) — итоги

| Вопрос из ТЗ | Ответ |
|---|---|
| (а) Дотянется ли демон до statusline-данных | **Да.** SDK: `query.usage_EXPERIMENTAL_…()` отдаёт 5ч/7д (% + resets_at), подписку, стоимость; `query.getContextUsage()` — заполнение контекста. Работает только на живом процессе → демон кэширует последние значения. Метод помечен экспериментальным — обёрнут в try/catch, при исчезновении просто пропадёт индикатор, не сломается демон |
| (б) Какие слеш-команды через SDK | **Все.** `supportedCommands()` вернул 53 (включая пользовательские скиллы). Локальные команды CLI (`/context`, `/compact`) тоже выполняются, проверено живьём |
| (в) Механика прикрепления файлов | Через API демона: `POST /api/upload` (base64) → файл в `~/.claude-pocket/uploads/`, путь дописывается в промпт; изображения дополнительно идут image-блоком в сообщение |
| (г) Архивные каталоги | Игнорировать. `-home-lazarevich` — дубликаты тех же UUID-сессий. Демон показывает только каталог своего cwd |

Прочее проверено живьём: `resume` продолжает **тот же** session_id (не форкает) — общие
сессии с CLI/VSCode работают; стрим-дельты (`includePartialMessages`) идут; `interrupt()`
работает (ход завершается `error_during_execution` + SDK бросает исключение — демон
это ловит и помечает ход «прерван»).

## Ключевые решения

1. **Демон на TypeScript-SDK / Node** (не Python): SDK всё равно спавнит Node-CLI, меньше слоёв. Чистый JS (ESM) без сборки.
2. **HTTP + SSE, не WebSocket**: SSE проще, автоматический reconnect с `Last-Event-ID`/`afterSeq`, буфер последних 500 событий в демоне.
3. **История из jsonl, не из своей БД**: транскрипты Claude Code — единственный источник правды; SQLite только для очереди ходов и настроек сессий. Формат записей: `user`/`assistant` (+ `ai-title` для заголовка), сайдчейны отфильтровываются.
4. **Жизненный цикл процессов**: один живой CLI-процесс на активный ход (streaming input mode — нужен для interrupt и usage). После ответа процесс живёт ещё 60 с (тёплое продолжение), потом закрывается — на машине 2 GB RAM пул постоянных процессов непозволителен. `maxConcurrent: 2`.
5. **Очередь в SQLite**: сообщение можно отправить когда угодно; если сессия занята — job ждёт; демон перезапустился — подвисшие running-джобы помечаются ошибкой.
6. **Авторизация API**: Bearer-токен в `~/.claude-pocket/token` (600). Приложение забирает его само по SSH (`cat`) — пользователь токен не вводит. Слушаем только 127.0.0.1.
7. **Новая сессия**: клиент шлёт `sessionId: null` → демон создаёт `new-<uuid>`, после `init` от SDK ремапит на реальный id (событие `session.created`), таблица алиасов в SQLite.
8. **Android без внешнего markdown**: свой мини-рендер (~120 строк) — заголовки/жирный/код/списки; меняется в одном файле.
9. **SSH-библиотека**: `com.github.mwiede:jsch` (живой форк) + BouncyCastle для ed25519-ключей. Порт-форвардинг `setPortForwardingL(0, …)` → OkHttp ходит на 127.0.0.1:<локальный порт>.
10. **Просмотр файлов**: `GET /api/file` (read-only, только внутри `$HOME`, до 2 МБ) — не SFTP: тот же канал, меньше кода. (В приложении экран просмотра — этап полировки.)
11. **Сборка под 2 GB RAM**: gradle без демона, `-Xmx1280m`, kotlin daemon 900m, minify выключен (не воюем с R8-правилами JSch/BC).
12. **Ключ подписи APK**: самоподписанный keystore в репо с паролем `claudepocket` — осознанно: это не публикация в маркет, а личная установка. Для публикации — перегенерить и убрать из репо.

## Безопасность SSH (добавлено 18-07-2026 по вопросу Алексея)

- **TOFU-проверка отпечатка сервера**: при первом подключении ключ хоста запоминается
  (`filesDir/known_hosts`), дальше `StrictHostKeyChecking=yes`. Смена ключа → понятная
  ошибка + кнопка «забыть старый отпечаток» (для случая переустановки сервера).
- **Автопрописка ключа устройства**: первый вход по паролю → приложение генерирует свою
  пару (ed25519, fallback RSA 3072) через JSch `KeyPair.genKeyPair`, дописывает публичный
  ключ в `~/.ssh/authorized_keys` (идемпотентно, с chmod 700/600), приватный сохраняет в
  настройках. Дальше вход идёт по ключу; пароль остаётся запасным путём.

## Дыры в безопасности (осознанные, легко переделать)

## Дистрибуция и обновления (19-07-2026, согласовано с Алексеем)

- **Хранение**: публичный GitHub-репозиторий `Cybiomez/claude-pocket`, APK — в GitHub Releases.
- **Версионирование**: semver из git-тега. `v0.2.0` → `versionName 0.2.0`, `versionCode 200`
  (major·10000 + minor·100 + patch). Без тега — `0.0.0-dev`. `CHANGELOG.md` в корне.
- **Сборка**: GitHub Actions (`.github/workflows/release.yml`) — пуш тега `v*` собирает,
  подписывает (keystore из секретов: `KEYSTORE_BASE64`, `KEYSTORE_PASSWORD`, `KEY_ALIAS`,
  `KEY_PASSWORD`) и публикует релиз с APK. Keystore в git не попадал (`*.keystore` в
  .gitignore с первого коммита); base64 для секрета — `~/Downloads/claude-pocket-keystore.base64`,
  бэкап — `~/Downloads/claude-pocket.keystore.backup` (терять нельзя — подпись обновлений).
- **Автообновление в приложении**: раз в сутки (или при принудительной проверке) запрос
  `api.github.com/repos/…/releases/latest`, сравнение semver, плашка на экране сессий →
  скачивание APK в кэш → системный установщик (FileProvider + REQUEST_INSTALL_PACKAGES).
  Подпись одна — ставится поверх.

## Сохранённые подключения (v0.3.0, ТЗ Алексея 19-07-2026)

- **Список вместо формы**: экран входа показывает сохранённые подключения; форма — по кнопке
  «Новое подключение» или карандашу (редактирование: имя, данные, удаление с подтверждением).
- **Хранение**: JSON-список в `filesDir/connections.enc`, шифрование AES-256-GCM, ключ в
  Android Keystore (alias `connections-store`, аппаратный, устройство не покидает).
  Старое открытое SharedPreferences-хранилище мигрирует первым элементом списка и очищается.
  Если ключ Keystore потерян (сброс/перенос на новое устройство) — расшифровать нечем,
  список начинается заново; это осознанно, бэкапа секретов нет по определению.
- **Разовый вход**: галочка «Запомнить подключение» снята → не сохраняем и не прописываем
  ключ устройства на сервер (`SshTunnel(enroll = false)`).
- **Порядок карточек**: перетаскивание (долгое нажатие → карточки покачиваются) реализовано
  вручную на Column + swap по половине высоты соседа; библиотеки reorder не тянули осознанно.
- **Приоритет ключа**: `PreferredAuthentications=publickey,password,keyboard-interactive` —
  раньше JSch мог выбирать пароль при живом ключе устройства.
- **Смена сервера/пользователя в редактировании** сбрасывает ключ устройства — к другому
  серверу он не подходит; при следующем входе по паролю пропишется новый.
- Проверка обновлений вручную: `UpdateChecker.check()` теперь пробрасывает сетевые ошибки,
  чтобы честно отличать «нет обновлений» от «нет сети».

## Самообновление через PackageInstaller (v0.3.5, запрос Алексея 19-07-2026)

- **Проблема**: ACTION_VIEW с «чужим» APK на каждое обновление собирал весь набор трения —
  системный установщик, разрешение «неизвестные источники», Play Protect с перекидыванием
  в Google Play. Плюс оборванная закачка давала невнятное «приложение не установлено».
- **Решение**: приложение ставит обновление само себе через `PackageInstaller` API
  (`setRequireUserAction(USER_ACTION_NOT_REQUIRED)`, разрешение
  `UPDATE_PACKAGES_WITHOUT_USER_ACTION`, статусы через `UpdateReceiver`). Первое такое
  обновление система подтверждает одним диалогом; после него приложение — «установщик
  записи» и на Android 12+ обновления идут без вопросов. При любой ошибке — откат на
  прежний ACTION_VIEW.
- Размер скачанного APK сверяется с Content-Length до установки; обрыв — понятная ошибка.
- Play Protect полностью «выключить» из приложения нельзя (сервис Google Play);
  этот путь просто не даёт ему поводов вмешиваться.
