# Журнал решений — Claude Pocket

Сессия 17-07-2026, автономная реализация пайплайна №4.

## Разведка (задача 2 пайплайна) — итоги

| Вопрос из ТЗ | Ответ |
|---|---|
| (а) Дотянется ли демон до statusline-данных | **Да.** SDK: `query.usage_EXPERIMENTAL_…()` отдаёт 5ч/7д (% + resets_at), подписку, стоимость; `query.getContextUsage()` — заполнение контекста. Работает только на живом процессе → демон кэширует последние значения. Метод помечен экспериментальным — обёрнут в try/catch, при исчезновении просто пропадёт индикатор, не сломается демон |
| (б) Какие слеш-команды через SDK | **Все.** `supportedCommands()` вернул 53 (включая пользовательские скиллы). Локальные команды CLI (`/context`, `/compact`) тоже выполняются, проверено живьём |
| (в) Механика прикрепления файлов | Через API демона: `POST /api/upload` (base64) → файл в `~/.claude-pocket/uploads/`, путь дописывается в промпт; изображения дополнительно идут image-блоком в сообщение |
| (г) Архивные каталоги | Игнорировать. `-home-lazarevich` — дубликаты тех же UUID-сессий. Демон показывает только каталог своего cwd |

Прочее проверено живьём: `resume` продолжает **тот же** session_id (не форкает) — общие
сессии с CLI/VSCode работают; стрим-дельты (`includePartialMessages`) идут; `interrupt()`
работает (ход завершается `error_during_execution` + SDK бросает исключение — демон
это ловит и помечает ход «прерван»).

## Ключевые решения

1. **Демон на TypeScript-SDK / Node** (не Python): SDK всё равно спавнит Node-CLI, меньше слоёв. Чистый JS (ESM) без сборки.
2. **HTTP + SSE, не WebSocket**: SSE проще, автоматический reconnect с `Last-Event-ID`/`afterSeq`, буфер последних 500 событий в демоне.
3. **История из jsonl, не из своей БД**: транскрипты Claude Code — единственный источник правды; SQLite только для очереди ходов и настроек сессий. Формат записей: `user`/`assistant` (+ `ai-title` для заголовка), сайдчейны отфильтровываются.
4. **Жизненный цикл процессов**: один живой CLI-процесс на активный ход (streaming input mode — нужен для interrupt и usage). После ответа процесс живёт ещё 60 с (тёплое продолжение), потом закрывается — на машине 2 GB RAM пул постоянных процессов непозволителен. `maxConcurrent: 2`.
5. **Очередь в SQLite**: сообщение можно отправить когда угодно; если сессия занята — job ждёт; демон перезапустился — подвисшие running-джобы помечаются ошибкой.
6. **Авторизация API**: Bearer-токен в `~/.claude-pocket/token` (600). Приложение забирает его само по SSH (`cat`) — пользователь токен не вводит. Слушаем только 127.0.0.1.
7. **Новая сессия**: клиент шлёт `sessionId: null` → демон создаёт `new-<uuid>`, после `init` от SDK ремапит на реальный id (событие `session.created`), таблица алиасов в SQLite.
8. **Android без внешнего markdown**: свой мини-рендер (~120 строк) — заголовки/жирный/код/списки; меняется в одном файле.
9. **SSH-библиотека**: `com.github.mwiede:jsch` (живой форк) + BouncyCastle для ed25519-ключей. Порт-форвардинг `setPortForwardingL(0, …)` → OkHttp ходит на 127.0.0.1:<локальный порт>.
10. **Просмотр файлов**: `GET /api/file` (read-only, только внутри `$HOME`, до 2 МБ) — не SFTP: тот же канал, меньше кода. (В приложении экран просмотра — этап полировки.)
11. **Сборка под 2 GB RAM**: gradle без демона, `-Xmx1280m`, kotlin daemon 900m, minify выключен (не воюем с R8-правилами JSch/BC).
12. **Ключ подписи APK**: самоподписанный keystore в репо с паролем `claudepocket` — осознанно: это не публикация в маркет, а личная установка. Для публикации — перегенерить и убрать из репо.

## Дыры в безопасности (осознанные, легко переделать)

- `StrictHostKeyChecking=no` — приложение не проверяет отпечаток сервера (MITM-риск).
  Правится в одном месте: `SshTunnel.kt`, TODO-метка. План: trust-on-first-use с сохранением отпечатка.
- Режим по умолчанию `bypassPermissions` + `allowDangerouslySkipPermissions` — агент не
  спрашивает разрешений. Меняется per-session через меню «Режим» или дефолт в
  `~/.claude-pocket/config.json`. План: проброс `canUseTool` в приложение с кнопками разрешить/запретить.
- Пароль/ключ SSH в SharedPreferences без шифрования (приватное app-хранилище).
  План: EncryptedSharedPreferences / Keystore.
